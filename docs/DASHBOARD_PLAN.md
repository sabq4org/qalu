# خطة عمل لوحة تحكم «ماذا قالوا؟»

> الإصدار الأول — 12 يوليو 2026. المرجع التنفيذي لتطوير `/dashboard`.

## أين نقف الآن

الموجود اليوم في `/dashboard`: شاشة واحدة (طابور المراجعة) تعرض التصريحات المعلقة بأزرار اعتماد/رفض/تصحيح إسناد + توثيق شخصية، خلف تسجيل دخول بدورين (admin/reviewer).

**الفجوة الجوهرية**: كل ما عدا المراجعة يُدار بسكربتات وSQL — إضافة شخصية، رفع صورة، تحرير سيرة، ترتيب العرض، تنظيف البيانات.

**المبدأ الحاكم**: كل عملية احتاجت psql أو سكربت يجب أن تصبح زراً في اللوحة.

---

## الأدوار والصلاحيات (أساس كل ما بعده)

| الدور | يفعل |
|-------|------|
| **admin** | كل شيء + إدارة المستخدمين + إعدادات الاستخراج والمصادر |
| **editor** (جديد) | إدخال يدوي + تحرير الشخصيات والسير والصور + كل صلاحيات المراجع |
| **reviewer** | اعتماد/رفض/تصحيح إسناد فقط — لا حذف ولا تحرير شخصيات |

## المرحلة 1 — «اللوحة تغني عن الطرفية» (الأولوية القصوى، ~أسبوع)

### 1.1 إدارة الشخصيات — `/dashboard/figures`
- جدول بكل الشخصيات (موثقة/غير موثقة) مع بحث وفرز وعدادات التصريحات بكل حالة.
- شاشة تحرير: الاسم، الصفة، السيرة، **رفع الصورة** (سحب وإفلات → R2 عبر presigned URL)، **ترتيب العرض البروتوكولي** بالسحب، توثيق/إلغاء توثيق.
- **دمج المكررات**: نقل كل تصريحات شخصية مكررة إلى الأساسية وحذف المكررة.
- حذف شخصية (تأكيد مزدوج، admin فقط).

### 1.2 الإدخال اليدوي الموثق — `/dashboard/statements/new`
- نموذج: الشخصية (بحث أو إنشاء من نفس النموذج)، النص الحرفي، المناسبة/السياق، التاريخ (ميلادي + عرض هجري)، الموضوع (اختيار أو جديد)، اسم المصدر، رابط المصدر، عنوان المقال.
- يُعتمد فوراً (إدخال محرر) مع تسجيل من أدخله — يقتل سكربتات seed نهائياً.

### 1.3 ترقية طابور المراجعة (الشاشة الحالية)
- فلاتر: الشخصية، الموضوع، درجة الثقة، المصدر + بحث نصي.
- **إجراءات جماعية**: تحديد متعدد → رفض/اعتماد دفعة كاملة.
- تحرير الموضوع والسياق قبل الاعتماد + حقل سبب الرفض (يُخزن لتدريب الفلاتر لاحقاً).
- اختصارات لوحة مفاتيح: A اعتماد / R رفض / أسهم للتنقل.
- عرض «المرفوضة» كأرشيف قابل للاسترجاع بدل الحذف.

### 1.4 إدارة المواضيع — `/dashboard/topics`
- قائمة بعدد تصريحات كل موضوع، إعادة تسمية، ودمج المرادفات («الاقتصاد» + «الشأن الاقتصادي»).

## المرحلة 2 — «غرفة عمليات الاستخراج» (~أسبوع)

### 2.1 شاشة مراقبة الاستخراج — `/dashboard/extraction`
- جدول `extraction_runs` بإحصاءات كل دفعة (مستخرج/مرفوض حرفياً/مكرر/فشل) + رسم زمني لنسبة الرفض الحرفي (ارتفاعها = تدهور جودة البرومبت).
- عداد تكلفة OpenAI التقريبية لكل دفعة.
- تحكم: **إيقاف/تشغيل الاستخراج** (flag في جدول settings يقرؤه الـ worker)، دفعة يدوية فورية، تعديل حجم الدفعة.
- **فلتر الاستهداف**: «شخصيات سعودية/خليجية فقط» يُحقن في برومبت الاستخراج — الحل الجذري لضجيج الأسماء الأجنبية.

### 2.2 إدارة المصادر — `/dashboard/sources`
- جدول `sources`: الصحيفة، رابط RSS، مفعلة/موقوفة، آخر سحب، إحصاء المستخرج منها.
- يبدأ بسبق ويستوعب الصحف السعودية (الرياض، عكاظ، واس...) عند تنفيذ أداة سحب الصحف.

### 2.3 سجل التدقيق — جدول `audit_logs`
- من فعل ماذا ومتى (اعتماد، رفض، تحرير، دمج، حذف) — إلزامي لمنصة توثيق.

### 2.4 إدارة المستخدمين — `/dashboard/users` (admin فقط)
- إضافة/تعطيل مستخدم، تغيير الدور، إعادة تعيين كلمة المرور.

## المرحلة 3 — «من أرشيف إلى منصة نشر» (~أسبوع)

### 3.1 مولد بطاقات السوشيال ⭐ الأعلى قيمة تسويقياً
- زر «إنشاء بطاقة» على أي تصريح معتمد → توليد صورة بقوالب الهوية (2d اقتباس 16:9، 2e مربع، 3a عاجل) → معاينة → تنزيل PNG أو نشر مباشر لحساب @mathaqalu في إكس.
- القوالب جاهزة في دليل الهوية (`docs/brand`) — المطلوب توصيلها بالبيانات فقط (نفس تقنية HTML→صورة المستخدمة في og.png).

### 3.2 لوحة نظرة عامة — `/dashboard` الرئيسية
- بطاقات: معلق للمراجعة، اعتمادات الأسبوع، شخصيات بلا صورة/سيرة، آخر دفعة استخراج، حالة الـ worker.

### 3.3 تحليلات خفيفة
- الأكثر زيارة من الشخصيات والتصريحات — ربط Cloudflare Web Analytics (مجاني وبلا كوكيز).

## البنية التقنية (ثوابت لكل المراحل)

- **المسارات**: `app/dashboard/{figures,statements/new,topics,extraction,sources,users}/page.tsx` خلف `DashboardLayout` + فحص دور لكل صفحة.
- **الأنماط**: الاستعلامات في `services/` فقط؛ الـ API تحت `app/api/admin/*` خلف `requireStaff()` (يتوسع لفحص الدور)؛ كل إجراء كتابة يبطل وسم الكاش المناسب (`revalidateTag`).
- **إضافات المخطط**: `audit_logs`، `sources`، `settings` (key/value)، دور `editor`، عمود `disabled` للمستخدمين — كلها `db:push` غير هدامة.
- **رفع الصور**: R2 عبر presigned URL.
- **UI**: مكونات اللوحة الحالية بألوان الهوية الدلالية (ذهبي = قيد التحقق، أحمر = رفض/حذف، تيل = اعتماد/مؤكد).
